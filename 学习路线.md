## 💡 学习建议

### 对移动端开发者的特别提示

1. **命名规范**：Python 使用 `snake_case`，不是 `camelCase`
2. **缩进**：Python 使用缩进而非大括号，通常是 4 个空格
3. **动态类型**：不需要声明变量类型，但可以使用类型提示
4. **集合类型**：Python 的列表、字典等非常强大，多加练习
5. **None vs null**：Python 使用 `None` 而非 `null` 或 `nil`

### 与移动端开发的对比

| 概念 | Swift/Kotlin | Python |
|------|--------------|--------|
| 变量声明 | `let/var` | 直接赋值 |
| 函数 | `func` | `def` |
| 类 | `class` | `class` |
| 继承 | `: ParentClass` | `(ParentClass)` |
| 空值 | `nil/null` | `None` |
| 布尔 | `true/false` | `True/False` |
| 字符串格式化 | `"\(var)"` | `f"{var}"` |
| 数组 | `Array/List` | `list` |
| 字典 | `Dictionary/Map` | `dict` |

### 学习节奏

- **每天 1-2 小时**
- **边学边练**：每个文件都有练习题
- **实践导向**：尽快开始写实际项目
- **利用经验**：很多概念你已经知道，只是语法不同
